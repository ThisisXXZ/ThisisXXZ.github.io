Search

关于 State Space Graph 与 Search Tree。

State Space Graph 是对某个搜索问题的数学建模 (mathematical representation)，它表示了所有状态与状态间的转移关系。Search Tree 则建立在 State Space Graph 之上，它的根节点是起始状态 (start state)，所有叶子节点都是目标状态 (goal state)，每一条从根到叶子的路径都代表一个可能的转移路径。

两种搜索风格 (taste)。

Tree Search Algorithm (TSA) 与 Graph Search Algorithm (GSA)。TSA 是不带 visit[] 数组的搜索，会重复访问节点。GSA 即带 visit[] 数组的搜索。一般附在某种具体搜索算法的后面，如 DFS-TSA 或 BFS-GSA 等。

盲目搜索 (Uninformed/Blind Search)。

  • 经典的 DFS (Depth-First Search) 与 BFS (Breadth-First Search)。此外还有一个很陌生的名词，UCS (Uniform-Cost Search)，它的策略是贪心的扩展当前 cost 最低的节点。定睛一看，这不就是 Dijkstra 嘛！
  • UCS，更准确的来说是 UCS-GSA，与 Dijkstra 算法的唯一区别是存在一个目标状态。而 Dijkstra 是单源最短路，只强调 source，不强调 sink。
  • 堆优化后的 Dijkstra 能达到 \(O(m\log{n})\) 的时间复杂度。虽然它能保证找到最短路，但仍然是盲目的，没有利用目标节点的位置信息。

知情搜索 (Informed Search)，或是我们熟悉的启发式搜索 (Heuristic Search)。

启发式函数 (Heuristic Function) \(h(x)\) 一般基于直觉或经验构建，它在可接受的开销内给出待解决的优化问题的一个可行解 (接近但不一定是最优解)。以寻路问题为例，不妨将其定义为当前状态到目标状态的直线距离或曼哈顿距离。

贪心搜索 (Greedy Search)：稍微更改一下 Dijkstra 中 PQ 的排序规则，\(h(x)\) 较低的节点拥有更高的扩展优先度。问题：直线距离确实更近了，但到门口时却发现有一堵墙。这是启发式函数的局限性带来的。与 Dijkstra (或 UCS-GSA) 相比，平均速度变快了，但无法保证解的 optimality。

A* 搜索是 Dijkstra 与贪婪搜索的结合。它吸收了 Dijkstra 中的 \(\text{distance(x)}\) 与贪心搜索中的 \(\text{heuristic(y)}\)。当在节点 \(x\) 扩展节点 \(y\) 时，计算 \(y\) 在 PQ 中的权重 \(f(y)=\text{distance(x)}+w_{x\to y}+\text{heuristic}(y)\)。在介绍完盲目的 Dijkstra 与短视的贪心搜索之后，这个想法应当是很自然的。

但是，Dijkstra 具有最优性 (optimality，即保证搜索得到的可行解是最优解)，贪婪搜索则不具有。为了使 A* 继承最优性，我们需要添加两个约束，从 TSA 到 GSA 约束由弱到强。

  • A*-TSA is optimal if an admissible heuristic is used.

    启发函数 \(h(x)\) 是 admissible 的当且仅当 \(0\leq h(x)\leq h^* (x)\)，其中 \(h^{*}(x)\) 是 \(x\) 到最近目标状态的实际距离。想像将最优路径上某点的 \(h(x)\) 设为无限大，该点将永远无法被扩展。而如果某点的 \(h(x)\) 较小，算法将向 Dijkstra 退化，最优化能够得到保证。详细证明见 Search.pdf pp.79。

  • A*-GSA is optimal if a consistent heuristic is used.

    启发函数 \(h(x)\) 是 consistent 的当且仅当对任意一条有向边 \(a\to c\)，都有 \(h(a)-h(c)\leq \text{cost}(a\to c)\)。它带来的结论是 \(f(a)\leq f(c)\)，也就是在任意一条路径上 \(f\) 是单调不减的。consistency 保证了首次被扩展的节点的路径就是到该节点的最优路径。

局部搜索 (Local Search)。

局部搜索解决的是 identification problem (识别问题)。与 planning problem 相对，这类问题只关心得到最终的目标状态，而不关心通向目标的路径。局部搜索评估并修改 (evaluate and modify) 当前状态，而不是从起始状态开始系统的 (systematically) 探索路径。例子：八皇后问题。

  • 状态 state
  • 后继状态 successors
  • 代价函数 cost function：若采用贪心的转移，选择代价函数最小的后继进行转移。

约束满足问题 (Constraint Satisfaction Problems, CSPs)。若干变量 (每个变量有对应 domain) [vertices]，若干约束 [arcs]。为变量赋值使得所有的约束都能被满足。解的 consistency 与 completeness。

解：Backtracking (回溯)，本质 DFS。为提高搜索效率，需要对搜索树 (变量的域) 进行剪枝 (pruning)。

  • Filtering & Forward Checking (FC). update domains to prune the search tree.
  • FC 的问题：只考虑值域，未考虑约束，无法提前发现某些必败局面。介绍 arc consistency：约束 \(x\to y\) 是 consistent 的当且仅当对 \(x\) (域) 的每个取值，\(y\) 都存在不违背约束的取值。注意当 \(x\) 失去某个值时，\(x\to y\) 可能变得 inconsistent。
  • Constraint Propagation. repeatedly enforces arc consistency (aka constraint consistency). e.g., AC-3 算法。传播体现在递归的过程：当 check \(x\to y\) inconsistent 时，pruning \(x\) 来维持约束；此时再递归地 check 所有指向 \(x\) 的约束。食用方法：每 assign 一个 value，进行一次 AC-3，对变量的可行域剪枝。若返回 true，继续搜索，返回 false，回溯。
  • AC-3 中的一个细节：若约束 \(x\to y\) inconsistent 且 \(x\) 被剪枝后，无需 check 约束 \(y\to x\) 是否 consistent。原因是对 \(x\) 剪枝已经保证了 \(x\)，\(y\) 的相容 (这真的是对 consistency 的一个最准确的翻译)。

进一步提高搜索效率的 trick：设置 variable ordering 与 value ordering。

  • variable ordering: choose which variable to assign values with? - fail-first heuristic
    • Minimum Remaining Values (MRV): the variable with fewest legal left values.
    • MRV 的 tie-breaker (Deg): the variable with the most constraints on remaining variables. degree heuristic
  • value ordering: choose which value to assign to a variable?
    • Least Constraining Value (LCV): the value that rules out the fewest values in the remaining variables

关于 fail-first heuristic 与 MRV：受到约束最多的变量也最容易耗尽所有可能的值从而引起回溯，所以应优先进行赋值。关于 LCV：尽量维持剩余变量面对约束的灵活性。

解澳大利亚省份染色问题的真 · 搜索算法：Backtracking + AC-3 + MRV & Deg + LCV。
