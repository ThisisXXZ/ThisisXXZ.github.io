Cryptographic primitives

Cryptographic primitives are well-established, low-level cryptographic algorithms that are frequently used to build cryptographic protocols for computer security systems.

In this course, 6 kinds of cryptographic primitives are covered:

  • Encryption scheme (private key encryption, public key encryption)
  • Message authentication code, MAC
  • Hash function
  • Key exchange protocol
  • Identification scheme
  • Digital signature

In most cases, the security of a cryptographic primitive is defined by an experiment (or game). The idea of reduction (归约) is critical in designing such experiments.

In different application scenarios, the definition of security may vary. For a cryptographic primitive, strong notions are often more secure than weak ones.

In constructing these primitives, many information theory and math related concepts are frequently applied. So we will also introduce some of them in P2:

  • Information theory
  • Pseudorandomness
  • One-way function (this is also some kind of a cryptographic primitive)
  • Number theory & Group theory
  • Hard problems (Factoring problem, RSA problem, Discrete logarithm problem, Diffie-Hellman problem)

These concepts ensure the correctness and security of some primitives.


Encryption scheme

Encryption scheme is the most common cryptographic primitive. (符合我对加密学的想象) It encrypts plaintext $m$ into ciphertext $c$. Without knowing specific key $k$, one cannot(*) recover $m$ from $c$.

Private key encryption

Private key encryption, also known as symmetric encryption, is the encryption scheme that uses only one secret key $k$. It consists of 3 algorithms: key generation algorithm $Gen$, encryption algorithm $Enc$, decryption algorithm $Dec$.

Algorithm $Gen(1^n)$ generates a secret key $k$. $k$ is only known by Alice (sender) and Bob (receiver).

A typical private key encryption application scenario:

  • $Gen(1^n)\to k$
  • Alice runs $Enc_k(m)\to c$
  • Alice transmits ciphertext $c$ to Bob through an insecure channel
  • Bob runs $Dec_k(c)\to m$, recovers the plaintext

Note that $Enc$, $Dec$ take both key $k$ and $m$/$c$ as input.

The unknowability of key $k$ is critical to the security of encryption scheme, because Kerckhoff’s principle indicates that the whole cryptosystem except the key is public knowledge. Adversary knows every detail about algorithms $Gen, Enc$ and $Dec$.

Public key encryption

Public key encryption, also known as asymmetric encryption, is the encryption scheme that uses 2 keys: public key $pk$ and private key $sk$. It also consists of 3 algorithms: $Gen, Enc$ and $Dec$.

Algorithm $Gen(1^n)$ generates $pk, sk$. Public key $pk$ is public knowledge; Private key $sk$ is only known by Bob (receiver). That’s why it is called asymmetric: the key distribution between 2 parties is uneven, Alice (sender) only knows $pk$, while Bob (receiver) knows both $sk$ and $pk$.

A typical public key encryption application scenario:

  • $Gen(1^n)\to \langle pk, sk\rangle$, $pk$ is distributed publicly. $sk$ remains secret except to Bob.
  • Alice runs $Enc_{pk}(m)\to c$
  • Alice transmits ciphertext $c$ to Bob through an insecure channel
  • Bob runs $Dec_{sk}(c)\to m$

Note that $Enc$ takes $pk, m$ as input, $Dec$ takes $sk, c$ as input.


Security of encryption scheme

Perfect Secrecy (Perfect indistinguishability)

$\Pr[M=m|C=c]=\Pr[M=m]$.

The strongest definition of security. It indicates that adversary with unbounded computational power cannot obtain any information about plaintext from a known ciphertext.

That is, knowing the corresponding ciphertext $c$ gives exactly no advantage to obtaining the plaintext $m$.

Shannon points out that an encryption scheme is secure only when $|\mathscr{K}|\geq|\mathscr{M}|$ ($\mathscr{K}$ is key space, $\mathscr{M}$ is message space). One-time pad is a typical perfectly secure scheme.

Perfect secrecy is an ideal definition; in real life, we relax this definition by:

  • Polynomial probabilisitic time (PPT) adversary (deprived of unbounded computational power)
  • adversary is allowed to success with negligible probability
