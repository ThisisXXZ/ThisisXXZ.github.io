What is Big Data?

Big Data Versus Small Data

To understand what big data is, we need to compare it with small data.

Small Data Big Data
small enough for human inference needs processing for human assumption
structured data stored in known forms such asJSON and SML could be structured, semi-structured or unstructured
accumulated slowly generated in large volumes, arrives continuously at enormous speed from multiple sources
mostly located in storage system with in Enterprises or data centers distributed on the cloud and server farms

Examples. Structured data: personal ID, address, phone number. Semi-structured data: social media posts with hash tags #. Unstructured data: pictures, videos.

Big Data Characteristics: 4V's

  • Velocity: Big Data is being generated extremely fast, and the process NEVER stops.
  • Volume: the amount of Big Data generated is vast.
  • Variety: Big Data is extremely diversified.
  • Veracity: the quality, origin, and conformity to facts and accuracy of the data.


Parallel Processing

Big Data needs parallel processing (in contrast to linear processing) due to capacity issues.

In parallelism, the problem statement is broken into a set of executable instructions. They are then distributed to multiple execution nodes of equal processing power and are executed in parallel.

  • Efficiency. Reducing processing time and memory.
  • Flexibility. Execution nodes can be added or removed when required.
  • Robustness. Errors can be fixed and executed locally independent of other instructions.

Data Scaling

Data scaling is a technique to manage, store, and process the overflow of data.

  • Scaling up: increasing the capacity of a single node.
  • Scaling out (horizontally): adding additional nodes with the same capacity until the problem is tractable. The most commonly used data scaling strategy.

The individual nodes arranged horizontally are called a computing cluster. The computing clusters can solve problems that are known as "embarrassingly parallel" calculations.

Embarrassingly parallel (or perfectly parallel) problems are the kind of workloads that can be easily divided and run independently of one another. If one process fails, it has NO impact on the others and can easily be rerun.

However, sorting a large data set adds significant complexity to the process, which makes the problem "not so easy parallel". Therefore the multiple computations must coordinate with one another because each process needs to be aware of of the state of its peer processes.

Basically, we are asking a cluster of computers to behave as a single computer. This requires sending messages across a network to each other or writing them into a file system that is accessible to all processes on the cluster.

Fault Tolerance

Fault tolerance enables a system to continue operating properly in the event of the failure of some of its components.

This works for Hadoop primary data storage system (HDFS) and other similar storage systems. Clearly, it is an extraordinarily complex maintenance process, but the Hadoop file system is a robust and time-tested framework, it can be reliable to 99.9%.


Big Data Tools and Ecosystem

There are six main components of Big Data tools, namely:

  • Programming tools: R, Java, Python, SQL, Scala, Julia...
  • Data Technologies: Hadoop, HDFS, Spark, MapReduce, Cloudera, Databricks...
  • Analytic and Visualization tools
  • Business Intelligence: Cognos, Pracle, PowerBI...
  • Cloud providers: Oracle, IBM...
  • NoSQL databases: HBase, MongoDB, CouchDB, Cassandra, Redis

We will mainly focus on data technologies, which allows enterprises to capture, process and share structured and unstructured data at any scale and in any format. They leverage high-performance, parallel Big Data processing.
